# Copilot Instructions

## ✅ Before You Do Anything
1. **Read `roadmap.md` first.**
   - Understand where the project currently is and what the next prioritized task is.
   - This is the source of truth for the current roadmap and phase.

## ✅ After You Complete an Action
1. **Update `roadmap.md`** with what was completed and what the next step should be.
   - Keep the roadmap current so the next command always has context.
   - If you changed priorities, add a short note in the relevant phase section.

---

## 🌀 Why This Matters
Keeping the roadmap up to date ensures that work remains aligned with the project vision, and it enables fast iteration using the “vibe coding” workflow.
